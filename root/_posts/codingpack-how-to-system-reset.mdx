# 코딩팩 시스템 초기화 및 복구

## 코딩팩에서 시스템 초기화

### 시스템 초기화란

시스템 초기화는 코딩팩의 핵심 프로그램인 `블록 드라이버`와 몇 가지 `필수 유틸리티`를 초기화합니다. 

<MdxTextBox>
<ul style={{margin:0}}>
  <li> 코딩팩을 사용하다가 예상치 못한 문제가 생긴 경우, 시스템 초기화가 필요할 수 있습니다.</li>
  <li> 또는 AI 코딩블록 사이트의 업데이트시 코딩팩의 시스템 초기화가 필요할 수도 있습니다.</li>
  <li> 시스템 초기화는 인터넷을 통해 다시 설치하는 방식으로 초기화합니다. 따라서 인터넷 연결이 필수입니다.</li>
</ul>
</MdxTextBox>


### 시스템 초기화 실행

시스템 초기화는 코딩팩 바탕화면의 `시스템 초기화 아이콘`을 더블 클릭하면 실행됩니다.

- 설치하기 전에 기존 설치된 버전은 자동으로 삭제된 후 진행됩니다.
- `삭제`하는 과정이 있으므로 주의가 필요합니다. 

<MdxImageView src="/images/codingpack/reset1.png" caption="바탕화면의 시스템 초기화 더블 클릭"/>

- 시스템 초기화를 실행하기 전에 인터넷 연결을 확인해주세요
- 초기화 실행은 보통 5~10분 정도 걸립니다.
- 네트워크 상태에 따라 시간이 더 걸릴 수 있습니다
- 초기화를 마친 후에는 자동으로 재부팅합니다.




## 시스템 복구(Rescue)

    <Box sx={{color:'secondary.main', fontWeight:600, mb:1}}>

    <Box>
    시스템 초기화 중에 문제가 생겼을 때, 보통은 한번 더 시스템 초기화를 실행하면 해결됩니다.
    하지만 시스템 초기화 중에 실수로 코딩팩의 전원을 끄는 경우가 간혹 있습니다. 
    이런 상황에서는 한번 더 실행해도 해결이 안되며 복구 명령을 실행하셔야 합니다. 
    </Box>
</MdxTextBox>

### 복구하는 방법

- 복구하는 과정은 터미널에서 이루어지는 것이라 조금 성가신 작업입니다. 
- 개발자들에겐 익숙하지만, 일반 사용자들에겐 조금 어려울 수 있습니다.

터미널을 열고 아래의 명령을 실행하면 됩니다. 

#### 복구 명령어 실행

```bash
curl https://raw.githubusercontent.com/aicodingblock/reset_blockdriver/release/blockDriver/rescue.sh | bash
```

위의 명령을 복사, 붙여넣기, 실행(엔터) 하면 되는데요.

#### TIP! 터미널에서 복사 붙여넣기

보통 복사할때 `CTRL + C`, `CTRL + V` 하시죠? 하지만, 터미널에서는 이 단축키를 이용할 수 없습니다.
- 터미널에서 복사는 `CTRL + INSERT`, 붙여넣기는 `SHIFT + INSERT` 입니다.
- 터미널에서 `CTRL + C`는 취소(cancel)입니다. 실행중인 명령을 중지할 때 사용합니다.
